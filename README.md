نظام إدارة المشفى (Hospital Management System):
1. فكرة المشروع
نظام لإدارة العمليات الطبية، سجلات المرضى، ومواعيد الأطباء بكفاءة عالية.

2. فريق العمل وتوزيع الأدوار
3. 
محمود السعدو: القيادة - مسؤول عن التأسيس والتوثيق.

محمد الأسعد: مطور - إدارة وحدة المرضى.

عبد الملك: مطور - إدارة وحدة المواعيد والأطباء.

3. التقنيات المستخدمة
اللغة: (HTML,CSS,JavaScript).

الإدارة: Jira & GitHub.

4. طريقة تشغيل المشروع
قم بتحميل ملفات المشروع (Download ZIP) أو عمل Clone للمستودع.

اذهب إلى المجلد الرئيسي للمشروع.

قم بفتح ملف index.html باستخدام أي متصفح إنترنت (مثل Google Chrome).

5. خطة العمل ومراحل التنفيذ (Work Plan)
لضمان جودة النظام وتسليمه في الوقت المحدد، تم تقسيم العمل إلى المراحل التالية:

المرحلة الأولى: التأسيس والتخطيط (Infrastructure & Planning)

إنشاء المستودع على GitHub وربط الفريق.

إعداد لوحة Jira وتوزيع المهام الأولية.

كتابة التوثيق الأساسي للمشروع (README).

المرحلة الثانية: تصميم الواجهات (UI/UX Design)

بناء الهيكل الإنشائي للموقع باستخدام HTML5.

تنسيق الواجهات والألوان باستخدام CSS3 لتناسب هوية المشفى.

توزيع العناصر (نموذج بيانات المريض، قائمة الأطباء، شاشة المواعيد).

المرحلة الثالثة: البرمجة والتفاعل (Front-end Logic)

إضافة التفاعل للواجهات باستخدام JavaScript.

برمجة عمليات التحقق من البيانات (Validation) داخل النماذج.

المرحلة الرابعة: المراجعة والدمج (Testing & Integration)

اختبار توافق الموقع مع المتصفحات المختلفة.

دمج الفروع (Merge Branches) ومراجعة الأكواد لضمان عدم وجود أخطاء.

6. تقسيم المهام (Task Distribution)
محمود السعدو: مسؤول عن التأسيس، إدارة Jira، وتنسيق ربط الصفحات.

محمد الأسعد: مسؤول عن تصميم وبناء واجهة إدارة المرضى (HTML/CSS).

عبد الملك: مسؤول عن برمجة الوظائف التفاعلية ونموذج المواعيد (JS).

7. شرح آلية العمل باستخدام Git
يتم العمل بنظام Branches، حيث لا يتم التعديل على الفرع الرئيسي main مباشرة.

يتم إنشاء فرع لكل ميزة جديدة (Feature Branch).

يتم استخدام Pull Requests لمراجعة التغييرات قبل دمجها.
